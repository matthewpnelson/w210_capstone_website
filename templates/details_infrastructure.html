Data

Our database contains information on over 65,000 different commercially produced beers from all over the world. Numerical data available for each beer includes the International Bitterness Units (IBU), the Alcohol by Volume (ABV), and an average beer rating based on popular beer rating platforms. Categorical data includes the beer Style, the producing Brewery name, and a detailed description for most beers (provided from the manufacturers marketing material. 

Cleaning 

Most of the beer data was full and complete, with ony a minor amount of data munging required before use in our predictive models. Beers missing a Rating were dropped (typically very small batch runs from small Breweries). Missing IBU's, and ABV's were imputed using median imputation, segregated by Beer Style to ensure congruency. 

Distribution of IBU by beer following a log transformation.

Distribution of ABV by beer following a log transformation.

Beer Ratings followed a normal distribution, with a noticeable difference between Breweries who produced less than 10 beers (orange) and those who produced more (blue), suggesting that there may be a learning curve for new breweries before they produce more widely enjoyable beers. 

The beer style variable was transformed from a categorical variable into a numerical variable through the use of scikit-learn's LabelEncoder() resulting in 168 different Beer Styles ready for ingestion into our model. 

In order to extract meaningful information (some qualitative information relating to ‘flavor’) from the descriptions, which do not follow any standardized form, the decision was made to utilize gensim's Word2Vec model to vectorize each description. Vectorization would allow us to establish similarities between words in each description, and calculate an average vector for the entire description which could then be included as multi dimensional variable input into the model. Instead of training our own Word2Vec model on only beer related vocabulary we leveraged Google News' pretrained word vector model which has each vector represented by 300 dimensions. Each word in the description was cleaned & tokenized, and then a vector was created for the entire description by simply averaging all of the individual word vectors within that description. 

Visual Outline of the Preprocessing Steps Taken.

Recommendation Model 

As this is an unsupervised machine learning task (there are no defined labels for each beer such as Recommend/Don't Recommend)we decided to utilize an unsupervised K-Nearest Neighbors model. The model was trained on the preprocessed data from above.

Model Validation 

Because the success/failure/accuracy of unsupervised machine learning tasks cannot be directly measured we decided to use a visual representation to establish whether or not the model performed as expected. Our multi-hundred dimensional feature space could not be visualized as-is, so we used t-SNE dimensionality reduction to bring our feature space down to two dimensions for easy plotting. Below is the resulting visualization showing all 65,000 beers in a two dimensional space. Color encoding was by beer style, suggesting that the model is able to establish nearest neighbors that would inherently be considered as similar due to their similarity in beer style. Said another way:  all of the beers with a style of 'IPA - American' are successfully grouped together, so that if you were to say you like a specific American IPA beer you would likely be recommended to try another American IPA beer with similar IBUs, ABV, rating, and description. 

t-SNE Dimensionality Reduced Plot Showing the Groupings of Beers by Beer Style

User Interaction

Users are able to interact with the pre-trained KNN model from two different perspectives. 

1) Specify an existing beer that they like currently and the KNN model uses the features of that beer to establish the nearest neighbors to that beer. This perspective required the use of a beer rating filter, since most users who like a beer would probably want to try a new beer that is considered 'better' than the beer they like while still retaining similar characteristics. Try it out here. 

2) Specify the exact parameters a user would like in a hypothetical beer. The KNN model then searches for the nearest neighbors to the hypothetical beer and recommends a few beers that already exist. This perspective is very powerful and highly unique to Alegorithm.io, as it can also incorporate a user written description as an input to the model. Try it out here.
